# 10. Resume Protocol

Если контекст сессии потерян, восстановление делаем строго в этом порядке:

1. Открыть `docs/tectonics/00_INDEX.md`.
2. Прочитать `docs/tectonics/01_REAL_PLATE_GEOMETRY.md`.
3. Прочитать `docs/tectonics/02_KINEMATICS_AND_ROTATIONS.md`.
4. Прочитать `docs/tectonics/04_OCEANIC_LITHOSPHERE_AND_BATHYMETRY.md`.
5. Прочитать `docs/tectonics/07_IMPLEMENTATION_PLAN.md`.
6. Проверить последнюю запись в `docs/tectonics/09_WORKLOG.md`.

## Перед любым кодом проверить

1. Модель изменения формы плит динамическая (не чистый Voronoi).
2. Типы границ считаются по `v_n/v_t`.
3. Океаническая глубина зависит от `ocean_age`, а не только от расстояния до берега.
4. Любой новый параметр имеет физический смысл и единицы.

## Что фиксировать в worklog после каждого шага

1. Какие поля добавлены.
2. Какие уравнения реально внедрены.
3. Какие метрики улучшились/ухудшились.
4. Какие регрессии появились.
